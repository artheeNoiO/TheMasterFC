# Deploy ไป Cloudflare (ไม่ใช้ Vercel)

## วิธี A — รัน Deploy-Cloudflare.bat (แนะนำ)

1. Double-click Deploy-Cloudflare.bat
2. Login Cloudflare ใน browser (ครั้งแรก)
3. รอ deploy เสร็จ
4. Dashboard → Workers & Pages → themasterfc → Custom domains
5. Add: www.themasterfc.com และ themasterfc.com

## วิธี B — อัปโหลดมือ (ถ้า bat ไม่ผ่an)

1. รัน Publish-Website.bat (build + เปิดโฟลเดอร์ dist)
2. Cloudflare → Workers & Pages → **แท็บ Pages** (ไม่ใช่ Workers)
3. Create → **Pages** → Upload assets
4. เปิด dist → Ctrl+A → ลากไฟล์ทั้งหมด (ต้องมี index.html + assets)
5. Project name: themasterfc (ไม่ใช่ Worker name แบบ random)
6. Deploy

## ถ้า Deploy ไม่ผ่an — เช็ค

- ต้องมี index.html ในรายการอัปโหลด
- ต้องมีโฟลเดอร์ assets (ไฟล์ .js .css)
- ชื่อ project ใช้ a-z 0-9 เท่านั้น เช่น themasterfc
- อย่าใช้หน้า "Create Worker" — ใช้ "Create Pages"
- ลองใหม่หลัง 10 นาที (Cloudflare บางที error ชั่วคราว)

## หลัง deploy สำเร็จ

https://www.themasterfc.com/
https://www.themasterfc.com/play

## โหมดออนไลน์ (Phase 1) — ตั้งครั้งเดียว

Cloudflare Pages ต้องมี secret เดียวกับ game API บน Render:

1. Double-click **Setup-Online-Bridge.bat** (สร้าง secret + ส่งขึ้น Cloudflare)
2. Render Dashboard → **the-socker-manager-api** → Environment
   - เพิ่ม `GAME_PROVISION_SECRET` = ค่าเดียวกับใน `server\.env`
3. รัน **Deploy-Cloudflare.bat** อีกครั้ง (ให้ Functions ใช้ secret ใหม่)

`GAME_API_URL` default เป็น `https://api.themasterfc.com` แล้ว — ไม่ต้องตั้งบน CF ถ้าใช้โดเมนนี้

Local dev: `server\.env` มี `GAME_PROVISION_SECRET` แล้ว — ไม่ต้องผ่าน CF session (localhost ใช้ JWT จาก :3001 โดยตรง)

## Home Server (API บนคอม — ฟรี)

1. **Setup-Home-Tunnel-Auto.bat** — login Cloudflare + สร้าง tunnel → `api.themasterfc.com`
2. **Start-Online-Stack.bat** — เปิด API + tunnel 2 หน้าต่างพร้อมกัน
3. ทดสอบ: https://api.themasterfc.com/health → `{"ok":true}`

ต้องเปิดคอม + tunnel ตลอด — ถ้าปิด เกมออนไลน์หยุด
