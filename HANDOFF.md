## ทำต่อ (คุยออกแบบไว้แล้วแต่ยังไม่เริ่มโค้ด — ใหญ่ทั้งหมด)

1. **ระบบดิวิชั่น 5-6 ชั้น + เลื่อนชั้น/ตกชั้นข้ามชาร์ด** — **เสร็จแล้ว** (`abf34c3`/`0f7c7eb`) `DIVISION_COUNT=6` ใน `game-version.js`, เปิดใช้จริง 2 ชั้นล่าง (`ACTIVE_DIVISIONS=[5,4]`) ชั้นบนสร้างอัตโนมัติเมื่อมีทีมเลื่อนขึ้นเป็นทีมแรก — `applyPromotionRelegation`/`relocateClubToDivision` ใน `gameService.js` ย้ายทีมข้ามชาร์ดด้วยเทคนิค "สลับ identity บนแถวบอท" (ไม่ย้าย shardId/id จริง กัน FK แมทเก่าพัง) เรียกจาก `startNewSeason` — ทดสอบผ่านสคริปต์ชั่วคราวแล้ว (ลบทิ้งหลังผ่าน)
2. **FM-style Tactics overhaul** — **ทำไปบางส่วนแล้ว** (`da7fdc1`, 2026-07-06) หลังสำรวจโค้ดพบว่า sandbox mode มีระบบ tactics ลึกอยู่แล้วเกินคาด: `MATCH_MENTALITIES` (5 ระดับ), `MATCH_INSTRUCTIONS` (กางเกม/บีบกลาง/บอลยาว/บอลสั้น), `TEMPO_OPTIONS`/`PRESSING_OPTIONS`/`DEF_LINE_OPTIONS` (การ์ดไอคอนแบบ FM26 อยู่แล้วใน `TeamStyleCards`), ระบบ role ต่อตำแหน่งละเอียดแบบ FM (`ROLE_DEFS`/`PLAYER_ROLES_BY_DPOS`/`resolvePlayerRole`) ที่ต่อเข้า simulation จริง (ไม่ใช่แค่ UI) ผ่าน `roleAttackDefenseMult`, และ offside trap/team talk/set-piece taker ครบแล้ว — **สิ่งที่เพิ่มรอบนี้:** dial ใหม่ "Creative Freedom" (`CREATIVE_FREEDOM_OPTIONS`, มีวินัย/สมดุล/อิสระ) ต่อเข้า `applyMatchPrepToContext` ทั้ง 2 จุด (pre-kickoff + live-match tick), จัด `TeamStyleCards`/`MatchPlanControls` ให้แบ่งเป็น 2 โซน "ครองบอล"/"ไม่ได้ครองบอล" ตามภาพ FM26 ที่ user ส่งมา — **ยังไม่ทำ (ใหญ่เกินสโคปรอบนี้):** ระบบ duty ต่อ role (Attack/Support/Defend modifier ซ้อนบน role เดิม แบบ "PF-At"/"IF-Su"), วิเคราะห์ positive/negative หลังแมท, และพอร์ตความลึกทั้งหมดนี้ไปที่ online match engine (`packages/game-engine`/`liveMatchService.js`) ซึ่งตอนนี้มีแค่ mentality อย่างเดียว ไม่มี tempo/pressing/defLine/creativeFreedom เลย
3. **ระบบรีเซ็ตรายเดือน + Battle Pass** — **เสร็จแล้ว** (`abf34c3`) รายละเอียดที่ user ยืนยัน: รีเซ็ตทุกต้นเดือน (เวลาไทย) = ล้างตารางคะแนนลีคทุกชาร์ด (ไม่แตะดิวิชั่น/สโมสร/นักเตะ) + Battle Pass ไม่ต้องรีเซ็ตมือ (key ด้วย seasonMonth "YYYY-MM" อยู่แล้ว) — XP จาก: ผลแมทลีค (ชนะ30/เสมอ15/แพ้5) + login รายวัน (10, กันซ้ำด้วย `lastLoginDay`) — 10 tier, reward = coin/cosmetic/staffCard (`BATTLE_PASS_TIERS` ใน `game-version.js`, `battlePassService.js`) — UI: เมนู More → "Battle Pass" (`BattlePassView` ใน `football-manager.jsx`) — **ยังไม่ทำ:** เควสรายวัน/รายสัปดาห์ (รอออกแบบเนื้อหา), ผูก reward "coin" เข้าระบบ Master Coin จริง (ยังไม่มีคอลัมน์เก็บยอด), `runMonthlyResetIfDue()` ยังไม่ได้รันจริงบน production (รอ deploy ครั้งถัดไปที่มี day-tick loop เรียกอัตโนมัติทุก 30s)
4. **[BACKLOG รอคุยสโคปเพิ่ม]** ระบบความสัมพันธ์นักเตะ (ยังไม่ชัดว่าหมายถึงเคมีทีม/เครือข่ายโซเชียลแบบ FM) + กราฟสถิติเชิงลึกแบบ FBref — user ส่งภาพอ้างอิงมาแล้ว (2026-07-05): radar chart เทียบผู้เล่น 2 คน (Overall/Passing/Shoot/Strength/Defense/Speed/Dribble), shot map+xG แบบ Thierry Henry chart, treemap กระจายสัดส่วนการยิงในทีม — เกมยังไม่เก็บข้อมูลตำแหน่ง/แรงกดดันรายนาที ต้องออกแบบ data layer ใหม่ทั้งชุดก่อนถึงจะทำได้จริง
5. Player Detail Modal + Radar Chart จริง — **เสร็จแล้ว** (คลิกไอคอน 📊 ข้างชื่อนักเตะใน Squad/Tactics) — **อัปเดต 2026-07-06:** เพิ่มโหมดเทียบผู้เล่น 2 คนซ้อนกราฟ (`PlayerDetailModal` รับ prop `squad`, มี dropdown เลือกคนเทียบ) ตอนนี้เปิดใช้แค่จุดที่เรียกจาก `TacticsSquadTable` (มี `squad` ในสโคปอยู่แล้ว) — จุดเรียกจาก `PlayerRow`/`SquadView` ยังไม่ได้ต่อ `squad` เข้าไป (ต้องส่งพร็อพลงมาหลายชั้น ข้ามไว้ก่อนเพื่อความเร็ว)
6. **[คุยไว้ ยังไม่ทำ]** นักเตะ "pool คงที่วนใช้" สำหรับ `genSquad()` — user อยากได้ 1,500 คนคงที่ (แทนสุ่มชื่อไทยไม่จำกัดทุกครั้ง) เพื่อให้ Cursor เจนรูปนักเตะแบบ 1-ครั้ง-ต่อคนได้ — **ยืนยันแล้วว่า `roster-database/rosters.generated.js` (2,944 คน, 252 legend, 8 ลีก × 368 คน, ชื่อ parody อิงโลกจริงทั้งหมด) ไม่ต้องแตะเลย** pool ใหม่ 1,500 คนเป็นคนละชุดแยกต่างหาก เฉพาะ `genSquad()` (Challenger League/บอท) — user บอก "เอาไว้คุยทีหลัง" ยังไม่เริ่มโค้ด
